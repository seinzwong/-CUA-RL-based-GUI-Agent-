# 飞书 CUA 项目结构

## 1. 项目定位
本项目是面向飞书桌面端差旅审批流程的 CUA 智能体实现，训练链路为：
SFT -> Exploration -> PRM -> Stepwise Reward -> PPO -> Evaluation。

## 2. 顶层结构

- assets/: 静态资源
- config/: PPO 超参数配置
- docs/: 路演模板、快速上手、提交清单
- eval/: 评测脚本与一键流水线
- eval_agent/: 任务定义、环境交互、代理调用
- exploration/: 轨迹探索脚本与输出
- ppo/: 强化学习训练与权重合并
- prm/: 进度估计训练、推理与数据组织
- sft/: SFT 数据构建与训练脚本

## 3. 关键模块

### 3.1 eval_agent
- envs/base.py: 环境抽象
- envs/feishu_desktop_env.py: 飞书桌面端 Mock 环境（含二次确认与回滚）
- tasks/base.py: 任务抽象
- tasks/feishu_travel.py: 飞书差旅任务
- configs/task/feishu_travel.json: 任务配置
- prompt/instructions/feishu_travel_inst.txt: 指令模板
- prompt/icl_examples/feishu_travel_icl.json: ICL 示例

### 3.2 sft
- build_feishu_sft.py: 生成飞书 SFT 种子数据
- data/feishu_travel_sft.json: 生成后的 SFT 数据
- feishu_travel_qwen3_8b.sh: SFT 启动脚本
- merge_lora.py: SFT LoRA 合并脚本

### 3.3 exploration
- feishu_travel/generate_response_feishu.py: 轨迹采样
- feishu_travel/run_explo_feishu.sh: 探索入口

### 3.4 prm
- data_org.py: 轨迹合并与格式整理
- train_our_progress_model*.py: PRM 训练脚本
- inference_prm.py: step-level 进度推理
- rl_data_org.py: PPO 训练数据构建

### 3.5 ppo
- step_ppo.py: PPO 主训练
- step_ppotrainer.py: 自定义 step-grained PPO Trainer
- train_ppo.sh: 训练入口
- merge.py: RL LoRA 合并

### 3.6 eval
- run_feishu_cua_pipeline.sh: 一键跑通全链路
- feishu_cua_eval.sh: 飞书评测入口
- run_eval.sh: 默认评测入口

## 4. 推荐执行顺序

1. `python sft/build_feishu_sft.py --input_file eval_agent/data/feishu_travel/train.json --output_file sft/data/feishu_travel_sft.json --replicas 8`
2. `bash sft/feishu_travel_qwen3_8b.sh`
3. `bash eval/run_feishu_cua_pipeline.sh`
4. `bash eval/feishu_cua_eval.sh`
